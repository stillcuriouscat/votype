## Codebase Patterns
- show_status() reads status/post_processor from DB — no IPC for state that DB tracks
- ensure_config_dir() deletes legacy files (PID, processing flag, audio path) on every startup
- is_process_running() removed — all callers now use DB-based state (is_recording, is_daemon_running)
- _persist/_restore_post_processor_id() removed — DB is sole persistence for post_processor
- POST_PROCESSOR_STATE_FILE removed — legacy migration handled by init_db() in state_db.py
- state_db functions (init_db, get_state, update_state) are imported at module level in voice_input.py
- STATE_DB_PATH constant in voice_input.py mirrors state_db.DEFAULT_DB_PATH — tests must patch BOTH
- ensure_config_dir() now calls init_db(STATE_DB_PATH) — any function calling ensure_config_dir() auto-initializes DB
- CLI state flow: update_state(status="recording") → update_state(status="processing") → update_state(status="idle")
- stop_recording reads get_state() snapshot BEFORE kill sequence — protects against concurrent state changes
- Tests using isolated_environment get automatic state DB initialization via conftest.py
- Tests that check is_daemon_running must also isolate DAEMON_LOCK_FILE (real daemon lock leaks otherwise)
- Daemon polls DB every 1s via _sync_status_from_db() in socket_server timeout loop — replaces IPC status commands
- Daemon __init__ reads post_processor from DB (not file) and validates against POST_PROCESSOR_PRESETS
- is_daemon_running() checks: flock → DB daemon_pid → PID file (in priority order)
- _make_daemon() helper in tests must include _current_db_status="idle" for __new__-constructed test instances
- ModelLoader: each framework has a static `load_<framework>_model()` method + `elif` in `load_model()` dispatch
- ModelInference: each framework has a static `transcribe_<framework>()` method + `elif` in `transcribe()` dispatch
- `load_model()` returns `(model, framework, extra_data)` tuple — extra_data is None when no extra state needed
- faster-whisper `model.transcribe()` returns `(Iterable[Segment], TranscriptionInfo)` — must unpack tuple
- Pre-existing test failures: qwen2-audio tests (model removed), soundfile not in test env
- Tests use `ASRDaemon.__new__(ASRDaemon)` to bypass __init__ — new attrs need explicit initialization in test fixtures
- post_processor_configs.py functions use lazy import of `voice_input.notify` to avoid circular dependency
- process_with_gemini_merge follows same SSH+vertex_proxy pattern as process_with_vertex_ai
- _run_vertex_proxy 429 fallback: replaces model in JSON stdin_data payload, reuses same cmd and timeout
