## Codebase Patterns
- ModelLoader: each framework has a static `load_<framework>_model()` method + `elif` in `load_model()` dispatch
- ModelInference: each framework has a static `transcribe_<framework>()` method + `elif` in `transcribe()` dispatch
- `load_model()` returns `(model, framework, extra_data)` tuple — extra_data is None when no extra state needed
- faster-whisper `model.transcribe()` returns `(Iterable[Segment], TranscriptionInfo)` — must unpack tuple

- Tests use `ASRDaemon.__new__(ASRDaemon)` to bypass __init__ — new attrs need explicit initialization in test fixtures
- post_processor_configs.py functions use lazy import of `voice_input.notify` to avoid circular dependency
- process_with_gemini_merge follows same SSH+vertex_proxy pattern as process_with_vertex_ai
