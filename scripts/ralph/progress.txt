## Codebase Patterns
- VOICE_INPUT_DATA_DIR is the base constant; MODELS_DIR and VOCAB_PATH derive from it
- Existing presets use "framework" key to determine which loader/pipeline branch to use
- Tests use sys.path.insert(0, project_root) for imports since project isn't pip-installed
- 3 pre-existing test failures in test_unit.py (daemon detection + stale default model assertion) — not related to ssh-claude work
- English regex in mixed CJK text: use `(?<![a-zA-Z])` / `(?![a-zA-Z])` instead of `\b` — Python treats CJK as `\w` so `\b` fails at CJK-English boundaries
- New module-level functions (load_vocab, apply_vocab, etc.) go before PostProcessorLoader class, after regex patterns
- Lazy import `from voice_input import notify` inside function body to avoid circular deps; mock target = `voice_input.notify`
- Lazy imports inside methods create LOCAL names — mock at the SOURCE module (e.g., `post_processor_configs.apply_vocab`), not the consuming module
