## Codebase Patterns
- POST_PROCESSOR_PRESETS dict in post_processor_presets.py: each entry has name, description, framework, and optional config
- Frameworks: "regex" (no config), "llama-cpp" (model_path, prompt_template, etc.), "firered-punc" (model_dir)
- MODELS_DIR = ~/.local/share/voice-input/models/ — all model files/dirs go here
- Venv activation: source ~/.local/share/voice-input/venv/bin/activate
- FireRedASR2S package: ~/code/FireRedASR2S (PYTHONPATH, not pip installed)
- PostProcessorLoader: static methods per framework (load_llama_model, load_firered_punc), routed by framework key in load_post_processor()
- FireRedPunc API: FireRedPunc.from_pretrained(model_dir, FireRedPuncConfig(use_gpu=False))
- Import: from fireredasr2s.fireredpunc.punc import FireRedPunc, FireRedPuncConfig
- FireRedPunc inference API: model.process([text]) → list of {"punc_text": ..., "origin_text": ...} (batch API, takes list)
- Case preservation: build {lowered: original} dict from re.findall(r'[a-zA-Z]+', text), lowercase input for model, re.sub to restore
- Launcher script: ~/.local/bin/voice-input (outside repo) — activates venv, sets PYTHONPATH, runs voice_input.py
- Deploy path: ~/.local/share/voice-input/ — symlinks to ~/code/voice_input/*.py; new .py files need symlinks added
- Daemon lifecycle: `voice-input kill` → `voice-input daemon` (allow ~20s for model loading before testing)
- E2E test: `source venv && PYTHONPATH=~/code/FireRedASR2S:$PYTHONPATH python test_e2e_kitty.py --verbose`
- E2E test with post-processor: `... python test_e2e_kitty.py --post-processor none --verbose`
- Auto-punctuation verification triggers automatically based on ASR model (firered-asr), not --post-processor flag
- Daemon socket commands: get_post_processor, set_post_processor (data: {post_processor_id: ...}), list_post_processors
- Daemon class is `ASRDaemon` (not VoiceInputDaemon) in voice_input.py
- punc_config.model_dir uses `~` prefix — must `os.path.expanduser()` before passing to loader
- MODEL_PRESETS['punctuation'] values: 'builtin' (ASR handles it), 'firered-punc' (needs FireRedPunc auto-load), 'none' (no punctuation)
- punc_config (sibling of config) only present when punctuation='firered-punc', contains model_dir
- Mosh/tmux Kitty windows: scrollback contains tmux border rendering (────), not clean transcription text — use daemon log as fallback verification
- _post_process pipeline is explicit: remove_fillers → auto-punctuation (punc_model) → LLM (post_processor_model) — don't call PostProcessorInference.process() from daemon
