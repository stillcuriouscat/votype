## Codebase Patterns
- Tests for proxy scripts go in `tests/test_spec_<name>.py`; mock the SDK in `sys.modules` via `patch.dict` and import the proxy lazily inside the patched block.
- `_run_vertex_proxy(cmd, stdin_data, timeout, fallback_model=None)` in `post_processor_configs.py` is proxy-script-agnostic — reuse for any new JSON-stdin SSH proxy.
- Vocab loading flag set: `("ssh-claude", "vertex-ai", "vertex-ai-merge", ...)` — extend tuple when adding new SSH frameworks (and the same gate in `_post_process`).
- Existing PP functions log success as `[SSH] <name> success: ...` and OpenRouter fallback as `[OPENROUTER] fallback success for <name>: <N> chars`.
- `notify(...)` is lazy-imported inside the functions (`from voice_input import notify`) to avoid circular import.
